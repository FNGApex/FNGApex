# FNGApex

I build tools that make other engineers faster.

Most of my work lives where software meets hardware: automation for lab
instruments, reverse-engineered vendor SDKs, concurrent pipelines that
replaced synchronous ones, and always-on systems running on
microcontrollers. If a machine ships with bad software, I treat that as an
invitation.

## What I work with

- **Daily drivers:** Python, C/C++
- **Also fluent:** Java, C#/.NET, Go, JavaScript, SQL
- **ML:** PyTorch, scikit-learn, XGBoost; training, evaluating, and
  repairing neural networks; agents built on heuristics, genetic algorithms,
  and reinforcement learning
- **LLM era:** RAG pipelines, agentic workflows, Claude Code as a power tool
- **Under the hood:** Linux, Docker, Redis, Git; transport protocols
  implemented from scratch (stop-and-wait, sliding window, TCP Reno)

## Things I have built

- Internal developer tooling that automated proprietary lab equipment, so
  researchers could run complex testing workflows without an engineer in the
  room. Error rates dropped 70% after I added automated validation and
  software locks.
- A concurrent testing architecture that replaced a synchronous pipeline and
  let physical test devices scale elastically.
- Three AI agents that play Tetris (heuristic search, a genetic algorithm,
  and reinforcement learning), evaluated against each other on lines
  cleared, survival time, and score consistency.
- An annotation-driven persistence framework in Java: custom annotations,
  dynamic proxies for lazy loading, Redis underneath.
- Reliable transport senders over raw UDP, then the measurements to prove
  which congestion strategy wins where.

## Current interests

Provably-correct neural network repair, sample-efficient optimization, and
LLM-driven developer workflows. The common thread: systems that can be
trusted to run unattended.
